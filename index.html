mod-01
Camada 2 / 3
Sniffing e
Interceptação
Captura passiva de pacotes com NIC em modo promíscuo. O driver descarta o filtro de MAC e repassa todos os frames ao sistema operacional, tornando visível tráfego de terceiros na mesma rede.
Wireshark 4.6.6 + Npcap  ·  Windows 10
Pacote capturado
# filtro: http — neverssl.com GET /online HTTP/1.1 Host: neverssl.com User-Agent: Mozilla/5.0 (Windows NT 10.0) Accept-Language: pt-BR Referer: http://neverssl.com/
Headers completos em texto claro — User-Agent, idioma, URL e Referer expostos sem qualquer criptografia
Consultas DNS também legíveis: google.com, discordapp.com, microsoft.com
mod-02
TLS / Handshake
Criptografia e
Handshake TLS
Servidor HTTPS local com certificado autoassinado. Criptografia híbrida: RSA/ECC para troca de chaves, AES-256-GCM para o tráfego. ECDHE garante Perfect Forward Secrecy.
Python 3 + trustme  ·  Wireshark Loopback
Handshake TLS 1.3
# filtro: tls — 127.0.0.1:4443 Client Hello SNI=localhost Server Hello TLS 1.3 selecionado Change Cipher Spec Application Data ← cifrado Application Data ← cifrado
Application Data completamente ilegível — contraste direto com o Módulo 1
NET::ERR_CERT_AUTHORITY_INVALID — certificado autoassinado não reconhecido por CA pública
mod-03
Camada de Aplicação
Gestão de Sessão
e Cookies
Aplicação Flask com autenticação por sessão. Demonstra a exposição do SessionID em HTTP puro e as consequências da ausência das flags Secure e SameSite.
Python 3 + Flask  ·  Wireshark Loopback
Cookie capturado
# filtro: http — POST /login Set-Cookie: session=eyJlc3VhcmlvIjoiUGVkcm8ifQ... HttpOnly; Path=/ # flags ausentes: Secure ← envia em HTTP sem criptografia SameSite ← vulnerável a CSRF
SessionID visível na rede — vulnerável a Session Hijacking sem HTTPS
HttpOnly presente — proteção contra leitura via JavaScript (XSS)
